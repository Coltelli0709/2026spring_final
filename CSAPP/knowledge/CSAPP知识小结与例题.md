## 第一章：计算机系统漫游

### 知识小结

- `●概念` 计算机硬件组件构成：CPU（含ALU、PC、寄存器文件）、主存、总线（数据总线、控制总线、地址总线）、I/O设备。
- `●概念` 程序计数器（PC/指令计数器）的作用：`存储将要执行的下一条指令在主存中的地址`。
- CPU 能直接存取内存储器中的数据，不能直接读取硬盘上的数据。
- `★重点` 存储设备的层次结构：寄存器 → L1 Cache → L2 Cache → L3 Cache → 主存 → 本地磁盘 → 远程存储。越往上越快、越小、越贵。
- `★重点` 书中配图涉及的总线结构和硬件名词不可忽略。

### 例题

**例1** IA32 的 CPU 中有一个程序计数器，它用于存储（ ）。

A. 保存将要执行的下一条指令在主存中的地址
B. 保存当前 CPU 所要访问的内存单元地址
C. 暂时存放 ALU 运算结果的信息
D. 保存当前正在执行的一条指令

答案：A

**例2** CPU 能直接读取硬盘上的数据吗？（判断）

答案：不能。CPU 只能直接存取内存储器，硬盘上的数据必须先加载到主存中。

## 第二章：信息的表示和处理

### 知识小结

#### 2.1 字节序（大小端）

- `◆计算` **小端法**（x86/Linux）：`低字节存在低地址，高字节存在高地址`。
- `◆计算` **大端法**：低字节存在高地址，高字节存在低地址。
- 例如 `int x = 0x01234567`，起始地址 `0x100`：小端法下 `0x100` 处存 `0x67`，`0x101` 处存 `0x45`...

**例3** 定义变量 `int x = 0x01234567;` 采用小端法存储，起始地址 `0x1000`，`x` 的存储形式为（ ）。

答案：`0x1000→0x67, 0x1001→0x45, 0x1002→0x23, 0x1003→0x01`

**例4** 使用小端法的机器中，`int x = 0x123456;` 语句的高地址字节是（ ）。

答案：`0x00`（int 是 4 字节，所以 `0x00 0x12 0x34 0x56`，高地址存 `0x00`）

#### 2.2 逻辑运算与位级运算

- `◆计算` `!`：逻辑非（结果只可能为 0 或 1）；`!!`：将非零值归一化为 1，零值保持 0。
- `◆计算` `&&` / `||`：逻辑与 / 或，返回值 0 或 1。
- `◆计算` `~`：按位取反；`&`：按位与；`|`：按位或；`^`：按位异或。
- `◆计算` 算术右移：左端补符号位；逻辑右移：左端补 0。
- `◆计算` 位移运算优先级低于加减法！

**例5** `!!0x21` 的结果为（ ）。

答案：`0x01`（0x21 ≠ 0，!0x21 = 0，!!0x21 = 1 = 0x01）

**例6** 对 `x = [10011101]₂`（8 位），`x >> 4`（算术右移）的结果是（ ）。

答案：`[11111001]₂`。算术右移左端补符号位 1。

**例7** 位级运算 `0x69 & 0x77` 的结果是（ ）。

答案：`0x61`（`0110 1001 & 0111 0111 = 0110 0001 = 0x61`）

#### 2.3 整数表示

- `◆计算` `Umax = 2^w - 1`，`Tmin = -2^(w-1)`，`Tmax = 2^(w-1) - 1`。
- `★重点` **有符号数与无符号数转换规则**：`位模式不变，数值重新解析`。
- `●概念` 有符号与无符号混合运算时，`有符号数隐式转换为无符号数`。
- `◆计算` 补码取负：`-x = ~x + 1`。`Tmin` 取负仍为自身（溢出）。
- `◆计算` `~x + ~y + 1 == ~(x+y)`（恒为真，因 `~x = -x-1`）。

**例8** 考虑 `short int x = -12345; unsigned short int y = x;` 则 y 的真值为（ ）。

答案：53191。`-12345` 的 16 位补码位模式解释为无符号数即 53191。

**例9** 32 位补码 `integer` 的最小值是（ ）。

答案：`-2³¹`（Tmin = -2147483648）

**例10** 下列代码执行后，哪些变量的值为 0？

```c
unsigned int a = 0xffffffff;
unsigned int b = 1;
unsigned int c = a + b;
unsigned long d = a + b;
unsigned long e = (unsigned long)a + b;
```

答案：`c` 和 `d`（32位溢出归零；d 的加法在32位完成再扩展）

**例11** 6 位字长，求值：

| 描述 | 值 |
|------|-----|
| Umax | 63 |
| Tmin | -32 |
| (unsigned)((int)-7) | 57 |
| (int)(20 + 12) | -32（6位补码溢出） |
| 12 && 4 | 1 |
| (!0x15) > 16 | 0 |

#### 2.4 浮点数

- `★重点` **IEEE 754 浮点格式**：`单精度(float): 1符号 + 8阶码 + 23尾数`，`双精度(double): 1符号 + 11阶码 + 52尾数`。
- `◆计算` **偏置量**：单精度 bias = 127（= `2^(8-1)-1`），双精度 bias = 1023。
- `★重点` **规格化数**：`E = exp - bias`, `M = 1 + frac`, `V = (-1)^S × M × 2^E`
- `★重点` **非规格化数**：`E = 1 - bias`, `M = frac`（用于表示 0 和极小数）
- `★重点` **特殊值**：阶码全 1 + 尾数全 0 = 无穷大；阶码全 1 + 尾数非 0 = NaN。
- `◆计算` **Round-to-Even（向偶数舍入）**：中间值向最低位为偶方向舍入。
- `●概念` `浮点加法满足交换律，不满足结合律`。
- `●概念` 大数吃小数：`(3.14 + 1e10) - 1e10 = 0.0`（3.14 被舍入丢失）。

**例12** 假设 8 位浮点数：1 sign, 3 exponent, 4 fraction。偏置量 = 3。

- 最小正浮点数：`0 000 0001` = `1/64`
- 最大正非规格化数：`0 000 1111` = `15/64`
- 负无穷大：`1 111 0000`
- NaN：`0 111 1111`（exp 全 1，frac 非 0）
- +∞：`0 111 0000`（exp 全 1，frac 全 0）

**例13** 8 位浮点数 (1,3,4)，值 9/32 的二进制表示和舍入值。

答案：`9/32 = 1.001₂ × 2^(-2)` → 位模式 `001 00`，舍入值 `1/4`（向偶数舍入向下）。

**例14** `(3.14 + 1e10) - 1e10` 在计算机中结果为（ ）。

答案：`0.0`（3.14 在对阶时被大数吞掉）

**例15** 判断真假：

| 表达式 | 是否恒真 | 反例 |
|--------|----------|------|
| `(x < y) == (-x > -y)` | 否 | x=Tmin, y=0 |
| `~x+~y+1 == ~(x+y)` | 是 | - |
| `(float)x == (int)(float)x` | 否 | x=Tmax |
| `d == (double)(float)d` | 否 | float 精度不足 |

## 第三章：程序的机器级表示

### 知识小结

#### 3.1 寄存器与寻址

- `★重点` **x86-64 寄存器命名**：`%rax`(64位) → `%eax`(低32位) → `%ax`(低16位) → `%al`(低8位)。32位前缀 `E`、64位前缀 `R` 需重点记忆。
- `%ebx` 引用的是 `%rbx` 寄存器的 `低32位`。
- `◆计算` **比例变址寻址**：`Imm(rb, ri, s)` = `M[Imm + R[rb] + R[ri]*s]`，其中 `s ∈ {1,2,4,8}`。

**例16** 假设 `%eax = 0x200`, `%ecx = 0x1`, `%edx = 0x3`，存储器数据如下表。求各操作数值。

| 存储器地址 | 值 |
|-----------|-----|
| 0x200 | 0xFF |
| 0x204 | 0xAB |
| 0x208 | 0x13 |
| 0x20c | 0x11 |

答案：

| 操作数 | 值 | 计算方式 |
|--------|-----|----------|
| `(%eax)` | `0xFF` | M[0x200] |
| `$0x204` | `0x204` | 立即数 |
| `0x1fc(,%ecx,4)` | `0xFF` | M[0x1FC + 1×4] = M[0x200] |
| `(%eax,%edx,4)` | `0x11` | M[0x200 + 3×4] = M[0x20c] |
| `1(%eax,%edx)` | `0xAB` | M[1 + 0x200 + 3] = M[0x204] |
| `516(%ecx,%edx)` | `0x13` | M[516 + 1 + 3] = M[0x208] |

#### 3.2 数据传送与栈操作

- `◆计算` `mov` vs `lea`：`mov 解引用地址（访问内存），lea 只做地址计算（不访问内存）`。
- `◆计算` `pushq %rbx`：`%rsp -= 8`，然后将 `%rbx` 的值存入 `(%rsp)`。
- `◆计算` `popq %rax`：读 `(%rsp)` 到 `%rax`，然后 `%rsp += 8`。
- `◆计算` x86-64 下 `pushq` 使 `%rsp` 减 8（64位 = 8字节）。

**例17** `mov` 和 `lea` 指令的区别是（ ）。

答案：`mov` 需要解引用地址，`lea` 不需要。

**例18** `%rsp = 0x108`，执行 `popq %rax` 后 `%rsp = ?`

答案：`0x110`（`%rsp += 8`）

**例19** `%rsp = 0xdeadbeefdeadd0d0`，执行 `pushq %rbx` 后 `%rsp = ?`

答案：`0xdeadbeefdeadd0c8`（`%rsp -= 8 = 0x...d0 - 8 = 0x...c8`）

**例20** 假设 `%eax = x`, `%ecx = y`，执行 `leal (%eax,%ecx,2),%edx` 后 `%edx = ?`

答案：`x + 2y`

#### 3.3 条件码与控制流

- `★重点` **条件码**：`CF`(进位), `ZF`(零), `SF`(符号), `OF`(溢出)。
- `◆计算` `cmp` 根据 `S2 - S1` 设置条件码；`test` 根据 `S1 & S2` 设置条件码。
- `◆计算` `jne` 在 ZF=0 时跳转（即 `test` 结果非零时）。
- `★重点` **switch 的跳转表实现**：`jmpq *Base(,%rax,8)`，其中 `%rax` 是 case 值索引。

**例21** 执行 `test %eax, %eax` 后 `jne 3d`，当 `%eax` 为（ ）时跳转。

答案：`1`（任何非零值）。`test %eax, %eax` 在 `%eax=0` 时 ZF=1；`jne` 在 ZF=0 时跳转。

**例22** switch 跳转表分析（见 2017-2018 期中第八题）：

给定跳转表地址表（索引0~7对应）：
```
[0]=0x4004d1  [1]=0x4004c8  [2]=0x4004c8  [3]=0x4004be
[4]=0x4004c1  [5]=0x4004d7  [6]=0x4004c8  [7]=0x4004be
```

分析：
- `0x4004c8` 对应 `default`（执行 `mov %esi,%eax; ret` → `result = y`）
- `0x4004be` 对应 `case 3/7`（`mov %edi,%eax; ret` → `result = x`）
- `0x4004c1` 对应 `case 4`（`mov $3,%eax; jmp 0x4004da` → `result = 3; 继续`）
- `0x4004d1` 对应 `case 0`（`mov %edi,%eax; and $0x19,%eax` → `result = x & 25`）
- `0x4004d7` 对应 `case 5`（`lea (%rdi,%rdi,1),%eax; add %esi,%eax` → `result = 2*x + y`）

#### 3.4 数组与结构体

- `●概念` `嵌套数组采用行优先 (Row-Major) 顺序排列`。
- `◆计算` 多维数组地址：`&A[i][j] = xA + (i * N + j) * sizeof(T)`。
- `★重点` **数据对齐目的**：`方便硬件 4 字节批量存取数据，提升内存读写效率`。
- `◆计算` **结构体对齐规则**：
  1. 每个成员偏移量必须能被其自身大小整除
  2. 结构体总大小必须能被最大对齐要求整除
  3. `char`(1), `short`(2), `int/float`(4), `long/pointer/double`(8)

**例23** x86-64 下结构体布局：

```c
struct my_struct {
    char a;           // 偏移 0
    long long b;      // 偏移 8（需8字节对齐，填充7字节）
    short c;          // 偏移 16
    float *d[2];      // 偏移 24（需8字节对齐，填充6字节）
    unsigned char e[3]; // 偏移 40
    float f;          // 偏移 44（需4字节对齐）
};
// 总大小需要被8整除 → 48字节后的第一个8倍数 = 48
```

计算：`a(1) + pad(7) + b(8) + c(2) + pad(6) + d(16) + e(3) + pad(1) + f(4) = 48`

**例24** 对上述结构体重新排序最少多少字节？

答案：48 字节。将 `long long` 和指针放前面：
```c
struct my_struct {
    long long b;        // 0-7
    float *d[2];        // 8-23
    float f;            // 24-27
    short c;            // 28-29
    char a;             // 30
    unsigned char e[3]; // 31-33
    // 总34，需被8整除 → 其实可以把大对齐成员放前面
};
```
实际上最优排列为：b(8) + d(16) + f(4) + c(2) + a(1) + e(3) = 34 → 对齐到 40。

**例25** 数组大小推导（2017-2018 期中第九题）：

汇编中 `array1` 索引计算 `36*i + 4*j` → `N * sizeof(int) = 36` → `N = 9`；`array2` 索引 `60*j + 4*i` → `M * sizeof(int) = 60` → `M = 15`。

#### 3.5 联合体 (Union)

- `◆计算` 联合体的所有成员共享同一块内存，大小为最大成员大小（对齐后）。
- `◆计算` 从不同的联合体成员视角读取相同底层位模式。

**例26** IA32 小端法下：

```c
union {
    unsigned char c[8];
    unsigned short s[4];
    unsigned int i[2];
    unsigned long k[1];
} dw;
for (j=0; j<8; j++) dw.c[j] = 0xc0 + j;
```

`dw.c`: `[c0, c1, c2, c3, c4, c5, c6, c7]`（低地址 → 高地址）

答案：
- `dw.s[0] = 0xc1c0`（小端：低地址c0低字节）
- `dw.s[1] = 0xc3c2`
- `dw.s[2] = 0xc5c4`
- `dw.s[3] = 0xc7c6`
- `dw.i[0] = 0xc3c2c1c0`
- `dw.i[1] = 0xc7c6c5c4`
- `dw.k[0] = 0xc3c2c1c0`

## 第六章：存储器层次结构

### 知识小结

- `★重点` **DRAM 核心概念**：
  - DRAM 依靠电容存储数据。
  - 电容会持续漏电，`断电数据丢失`。
  - `需要持续供电 + 定期刷新` 来维持电容状态。
  - 刷新机制：周期性读取并重写每个存储单元。
- `●概念` RAM（随机存储器）：SRAM（快、贵、用作Cache） vs DRAM（慢、便宜、用作主存）。
- `◆计算` 磁盘容量 = `盘片数 × 2面 × 磁道数/面 × 扇区数/磁道 × 字节数/扇区`。
- `◆计算` 扇区访问时间 = `寻道时间(Tavg_seek) + 旋转延迟(Tavg_rotation) + 传送时间(Tavg_transfer)`。
- `●概念` SSD特点：更快速、低能耗；但 `写访问比读访问慢`，读写速度不相当。

**例27** 5 个盘片，每扇区 512 字节，每面 20000 磁道，每磁道 400 扇区，磁盘容量？

答案：`5 × 2 × 20000 × 400 × 512` = `40.96 GB`

**例28** 15000 RPM, `Tavg_seek = 8ms`, 每磁道 500 扇区，扇区访问时间？

答案：旋转延迟 = `(60/15000)/2 = 2ms`；传送时间 = `60/15000/500 = 0.008ms`；总 = `8 + 2 + 0.008 ≈ 10ms`

**例29** 不是 SSD 特点的是（ ）。

答案：读写速度相当（SSD 写比读慢很多）

**例30** 局部性原理：正确的是（ ）。

答案：局部性包括时间局部性和空间局部性。步长越大，空间局部性越差。

## 第七章：链接

### 知识小结

- `●概念` 链接器主要任务：`符号解析 + 重定位`。
- `★重点` **静态库链接规则**：命令行从左到右扫描，每个库解析前面所有目标文件和库中积累的未定义符号。如果库之间有循环依赖，需要重复写出库名。
- 目标文件：可重定位目标文件(.o)、可执行目标文件、共享目标文件(.so)。

**例31** 依赖关系场景，给出最小命令行：

(1) `p.o → libx.a → p.o`（循环依赖，但 p.o 先出现）

答案：`gcc p.o libx.a`（p.o 先扫描，引用 libx 符号；libx 引用的 p.o 符号已在前面解析）

(2) `p.o → libx.a → liby.a` 且 `liby.a → libx.a`

答案：`gcc p.o libx.a liby.a libx.a`（libx.a 需出现两次以解析 liby.a 引用的符号）

(3) `p.o → libx.a → liby.a → libz.a` 且 `liby.a → libx.a → libz.a`

答案：`gcc p.o libx.a liby.a libx.a libz.a`

## 第八章：异常控制流

### 知识小结

#### 8.1 进程

- `●概念` `fork()` 函数：`调用一次，返回两次`。父进程返回子进程 PID（>0），子进程返回 0。
- `●概念` 子进程获得父进程的独立副本（写时复制），包括独立的地址空间。
- `◆计算` fork 后全局变量的修改互不影响。

**例32** 下列程序输出是什么？

```c
int counter = 1;
int main() {
    if (fork() == 0) {
        counter--;
        exit(0);
    } else {
        wait(NULL);
        printf("counter = %d\n", ++counter);
    }
    exit(0);
}
```

答案：`counter = 2`。子进程修改自己的 counter 副本（变为 0）后退出；父进程 wait 后 `++counter`（1 → 2）。

#### 8.2 信号（本次新增重点）

- `★重点` **信号发送与接收**：发送方（内核/其他进程）→ 经内核转发 → 目标进程。
- `★重点` **信号处理方式**：默认程序处理 / 自定义信号处理函数。
- `★重点` **未决信号**：`已发送但进程尚未接收处理的信号`。
- `★重点` **同类信号排队阻塞**：进程正在处理某类信号时，同类新信号排队等待，不会覆盖。
- `★重点` **信号屏蔽**：可通过函数手动设置阻塞；取消阻塞后排队信号才会被处理。
- `★必考` **进程两位向量**：`未决向量 (Pending)` 和 `阻塞向量 (Blocked)`。每个位对应一个信号编号：`1 = 未决/阻塞状态，0 = 无此状态`。
- `★必考` **信号处理上下文切换**：
  - 普通跳转（`setjmp`/`longjmp`）：只保存栈、参数、缓冲区数据，`不保存信号屏蔽状态`。
  - **信号专用跳转接口**：必须用于信号处理程序中。会同步保存当前信号屏蔽向量，恢复运行时还原屏蔽状态，`避免信号屏蔽异常和信号丢失`。

**例33** 什么是未决信号？

答案：已由发送方发出、经内核传递到目标进程、但目标进程尚未接收和处理的信号。

**例34** 进程正在处理 SIGINT 信号时，又一个 SIGINT 到达，会发生什么？

答案：新的 SIGINT 信号会排队等待，不会覆盖原有的待处理信号。同类信号按排队方式处理。

**例35** 为什么信号处理函数中不能用普通跳转（longjmp），而必须用信号专用跳转接口？

答案：普通跳转不会保存和恢复信号屏蔽向量。如果信号处理函数中改变了屏蔽状态（如阻塞某信号），用普通跳转返回后屏蔽状态会异常保留，导致后续信号接收异常和信号丢失。专用接口会同步保存和恢复屏蔽向量。

## 第九章：虚拟内存

### 知识小结

#### 9.1 地址翻译与 TLB

- `●概念` **页表作用**：将虚拟页号（VPN）映射到物理页号（PPN）。
- `★重点` **TLB**：加速虚→实地址变换的小型 Cache。
- `◆计算` **虚地址结构**（32位为例）：
  - VPO（页内偏移）= 物理页内偏移 PPO
  - VPN（虚页号）= 高位
  - VPN 再分为 TLBT（Tag）+ TLBI（组索引）
- `◆计算` 物理地址 PA = `{PPN, PPO}`

**例36** 虚地址 32 位，VPN 24 位，PPN 32 位，TLB 2路组相联 512 条目（2017-2018 期末第八题）：

(1) 地址位范围：
- VPO: `0-7`（8位 = 页面大小 256B）
- VPN: `8-31`（24位）
- PPO: `0-7`（= VPO）
- PPN: `8-39`（32位）
- TLBI (TLB组索引): VPN 的 `0-7` 位（512条目 / 2路 = 256组 = 2^8 → 8位）
- TLBT (TLB标记): VPN 的 `8-23` 位

(2) 虚拟地址 `0x083F3E9A`：
- VPN = `0x083F3E`（取高24位：`0x083F3E9A >> 8`）
- VPO = `0x9A`（取低8位）
- TLBI = `0x3E`（VPN 低8位）
- TLBT = `0x083F`（VPN 高16位）
- 查 TLB 表：TLBI=`0x3E` 的第二路有 Tag=`0x083F`, Valid=1 → 命中！PPN = `0xAB18ED24`
- PA = `{PPN, PPO}` = `0xAB18ED249A`

(3) 虚拟地址 `0x083E409B`：
- VPN = `0x083E40`, VPO = `0x9B`
- TLBI = `0x40`, TLBT = `0x083E`
- 查 TLB 表：TLBI=`0x40` 第一路 Tag=`0x083E` 但 Valid=0；第二路 Tag=`0x3E40` 不匹配 → 不命中！PPN 未知

#### 9.2 内存分配（分值占比高）

- `★重点` **堆内存动态分配**：`malloc()` 分配堆空间，`free()` 释放。堆在运行时可以动态扩展和收缩。
- `★重点` **内存分配结构体** → 需熟记（含头部/脚部的块大小、分配位等字段）。
- `◆计算` **空闲块合并规则**：分配/释放内存时，检查前后相邻内存块是否空闲；满足条件则合并相邻空闲块，并修改块长度等元数据。
- `◆计算` **空闲块检索方式**：
  - 首次适配 (First Fit)：找到第一个足够大的空闲块
  - 下次适配 (Next Fit)：从上次分配位置继续搜索
  - 最佳适配 (Best Fit)：找到最小的满足要求的空闲块

**例37** 当调用 `malloc` 这样的 C 标准库函数时，（ ）可以在运行时动态扩展和收缩。

答案：堆 (Heap)

**例38** 什么是空闲块合并？何时触发？

答案：当一块内存被释放时，分配器检查其物理上相邻的前后内存块是否也处于空闲状态。如果相邻块空闲，则将其合并为一个更大的空闲块，同时更新块头部和脚部的长度信息。合并可在 `free()` 时立即执行，也可延迟到后续分配时触发。

**例39** 三种空闲块检索方式各有什么优缺点？

答案：首次适配简单快速但可能在前端产生碎片；下次适配分布更均匀但可能降低局部性；最佳适配碎片最小但搜索开销大。

## Cache 与性能

### 知识小结

- `◆计算` Cache 结构参数：`S`（组数）、`E`（每组行数/相联度）、`B`（块大小）。
- `◆计算` 地址划分：
  - CO（块内偏移）: `log₂(B)` 位
  - CI（组索引）: `log₂(S)` 位
  - CT（标记位）: `地址总位 - CO - CI`
- `◆计算` 不命中率计算：冷/冲突/容量不命中的区分。

**例40** Cache 大小 128 字节，2 路组相联，块大小 32 字节（2017-2018 期末第七题）：

(1) 最多保存多少数组元素？答案：`128/8 = 16`（double 占 8 字节）

(2) 有多少组？`128/(2×32) = 2` 组

(3) `m=1, n=32` 时（顺序访问）：不命中率 = `1/4`（每4个 double 一个冷不命中，然后 3 个命中）

(4) `m=2, n=16` 时（步长 16）：不命中率 = `1/2`

(5) `m=16, n=2` 时（步长 2）：不命中率 = `1/4`；具有空间局部性

**例41** 根据汇编代码填写缺失的 C 代码（2017-2018 期中第五题）：

```asm
mystery1:
    pushq %rbx
    movq %rdi, %rbx       # rbx = x
    movl $0, %eax          # result = 0
    testq %rdi, %rdi       # test x
    je .L2                 # if x==0, return 0
    shrq $2, %rdi          # nx = x >> 2
    call mystery1          # rv = mystery1(nx)
    addq %rbx, %rax        # return x + rv
.L2:
    popq %rbx
    ret
```

对应 C 代码：

```c
long mystery1(unsigned long x) {
    if (x == 0) return 0;
    unsigned long nx = x >> 2;
    long rv = mystery1(nx);
    return x + rv;
}
```

### 安全漏洞类

**例42**（2015-2016 期中第四题）分析 `memcpy` 的 `size_t` 漏洞：

```c
int copy_from_kernel(void *user_dest, int maxlen) {
    int len = KSIZE < maxlen ? KSIZE : maxlen;
    memcpy(user_dest, kbuf, len);
    return len;
}
// 调用：copy_from_kernel(mybuf, -MSIZE);
```

问题：`maxlen` 为负数（`int`），赋给 `len` 后仍为负数；传入 `memcpy` 时转为 `unsigned int`（超大正数），导致内存越界复制。

修复：将 `maxlen`、`len`、返回类型都改为 `unsigned int`。
