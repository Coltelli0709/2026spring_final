# CSAPP 期末复习 — 知识框架树状图

> 标注说明：`★` = 老师划重点 / 高频考点，`◆` = 计算题核心，`●` = 概念简答题核心

```
CSAPP
├── 第一章：计算机系统漫游
│   ├── ◆ 软件评估流程
│   ├── 计算机硬件组件
│   │   ├── 数据总线 (Data Bus)
│   │   ├── 控制总线 (Control Bus)
│   │   ├── 地址总线 (Address Bus)
│   │   └── I/O 总线
│   ├── 存储设备的层次结构
│   └── ★ 书中配图相关知识点（不可忽略偏门配图、硬件名词）
│
├── 第二章：信息的表示和处理
│   ├── 信息存储
│   │   ├── ◆ 字节序：`大端法 (Big Endian)` vs `小端法 (Little Endian)`
│   │   ├── ◆ 逻辑运算：`!` `!!` `&&` `||` `~` `&` `|` `^`
│   │   ├── ◆ 位移运算：`算术右移`（补符号位）vs `逻辑右移`（补0）
│   │   └── 位级运算与掩码操作
│   │
│   ├── 整数表示
│   │   ├── 无符号编码 (Unsigned): `Umax = 2^w - 1`, `UMin = 0`
│   │   ├── 补码编码 (Two's Complement): `Tmin = -2^(w-1)`, `Tmax = 2^(w-1) - 1`
│   │   ├── ◆ 有符号/无符号转换规则：`位模式不变，数值重新解析`
│   │   ├── ◆ `补码加减法运算`（核心计算考点）
│   │   ├── ◆ 溢出判断与截断
│   │   └── ★ C语言隐式类型转换：`有符号+无符号 → 无符号`
│   │
│   ├── 整数运算
│   │   ├── ◆ 无符号加法溢出
│   │   ├── ◆ 补码加法溢出（正溢出 & 负溢出）
│   │   ├── 乘法与移位优化（`<<` 代替 `*2^k`）
│   │   └── 除法与移位（算术右移向负无穷舍入）
│   │
│   └── 浮点数
│       ├── ★ IEEE 754 浮点格式
│       │   ├── `单精度 (float, 32位): 1符号 + 8阶码 + 23尾数`
│       │   └── `双精度 (double, 64位): 1符号 + 11阶码 + 52尾数`
│       ├── ★ 规格化/非规格化/特殊值
│       │   ├── 规格化: `E = exp - bias`, `M = 1 + frac`
│       │   ├── 非规格化: `E = 1 - bias`, `M = frac`
│       │   ├── 无穷大: `exp全1, frac全0`
│       │   └── NaN: `exp全1, frac非0`
│       ├── ◆ Round-to-Even（向偶数舍入）规则
│       ├── ● `浮点加法满足交换律，不满足结合律`
│       └── ● 大数吃小数问题：`(3.14 + 1e10) - 1e10 = 0.0`
│
├── 第三章：程序的机器级表示
│   ├── 程序编码
│   │   ├── ◆ `mov` vs `lea` 指令区别
│   │   │   └── ● `mov 需要解引用地址，lea 仅做地址计算`
│   │   └── x86-64 寄存器命名规律
│   │
│   ├── 数据格式与寻址
│   │   ├── ★ `寄存器前缀：32位(E), 64位(R)` → 重点记忆
│   │   ├── ◆ 操作数格式
│   │   │   ├── 立即数: `$Imm`
│   │   │   ├── 寄存器: `Ra`
│   │   │   ├── 存储器: `Imm(rb, ri, s)` = `M[Imm + R[rb] + R[ri]*s]`
│   │   │   └── 比例变址寻址: `(base, index, scale)`
│   │   ├── ◆ 数据传送指令：`movb/movw/movl/movq`, `movzbw/movsbl` 等
│   │   └── ◆ 栈操作：`pushq`（`%rsp先减再存`）, `popq`（`先读再加`）
│   │
│   ├── 算术与逻辑操作
│   │   ├── `leaq`: 加载有效地址（不访问内存）
│   │   ├── 一元操作: `inc`, `dec`, `neg`, `not`
│   │   ├── 二元操作: `add`, `sub`, `imul`, `xor`, `and`, `or`
│   │   └── 移位: `sal`, `shl`, `sar`, `shr`
│   │
│   ├── 控制
│   │   ├── ★ 条件码寄存器
│   │   │   ├── `CF`（进位标志）
│   │   │   ├── `ZF`（零标志）
│   │   │   ├── `SF`（符号标志）
│   │   │   └── `OF`（溢出标志）
│   │   ├── ◆ `cmp` 与 `test` 指令（只设条件码，不修改寄存器）
│   │   ├── ◆ 条件跳转：`je/jne/jg/jl/ja/jb` 等
│   │   ├── ◆ `switch` 语句的跳转表实现
│   │   │   └── `jmpq *Base(,%rax,8)` → 跳转表中的索引访问
│   │   └── ★ 循环的汇编实现（条件跳转 + 跳回）
│   │
│   ├── 过程调用
│   │   ├── ★ 栈帧结构：`%rbp`（基址）, `%rsp`（栈顶）
│   │   ├── ◆ `call`（压返回地址+跳转）, `ret`（弹出返回地址+跳转）
│   │   ├── 参数传递：前6个整数参数 → `%rdi,%rsi,%rdx,%rcx,%r8,%r9`
│   │   └── ★ 调用者保存 vs 被调用者保存寄存器
│   │
│   └── 数组与结构体
│       ├── 数组
│       │   ├── ● `嵌套数组采用行优先 (Row-Major) 顺序排列`
│       │   └── ◆ 多维数组地址计算：`&A[i][j] = A + (i*N + j) * sizeof(element)`
│       └── 结构体与联合体
│           ├── ★ `数据对齐`：对齐目的是 `方便硬件4字节批量存取，提升内存读写效率`
│           ├── ◆ `结构体布局与对齐规则`
│           │   ├── 每个成员偏移量必须能被其大小整除
│           │   └── 结构体总大小必须能被最大对齐成员大小整除
│           └── ◆ 联合体 (Union)：所有成员共享同一存储空间
│
├── 第六章：存储器层次结构
│   ├── 存储技术
│   │   ├── RAM（随机存储器）
│   │   ├── ★ `DRAM核心概念`
│   │   │   ├── ● `DRAM依靠电容存储数据，电容持续漏电，断电数据丢失`
│   │   │   └── ● `刷新机制：需持续供电+定期刷新以维持电容状态`
│   │   ├── 传统磁盘
│   │   │   ├── ◆ 磁盘容量计算: `盘片数 × 2面 × 磁道数 × 扇区数 × 每扇区字节数`
│   │   │   └── ◆ 扇区访问时间 = `寻道时间 + 旋转延迟 + 传送时间`
│   │   └── ● SSD特点：更快速、低能耗、`写比读慢`，读写速度不相当
│   │
│   └── 局部性原理
│       ├── ● `时间局部性`：重复引用同一变量
│       ├── ● `空间局部性`：引用相邻地址
│       └── ● `步长越大，空间局部性越差`
│
├── 第七章：链接
│   ├── 链接器任务：符号解析 + 重定位
│   ├── ★ 静态库链接规则
│   │   └── ◆ `命令行从左到右扫描，后出现的库解析前面积累的未定义符号`
│   │       └── 有循环依赖时需重复写出库名
│   └── ● 目标文件类型：可重定位、可执行、共享目标文件
│
├── 第八章：异常控制流
│   ├── 进程基础
│   │   ├── ◆ `fork()`: `调用一次，返回两次`
│   │   │   ├── 父进程返回子进程PID
│   │   │   └── 子进程返回0
│   │   ├── 并发与并行
│   │   └── ◆ `wait()` 回收子进程
│   │
│   └── ★ 信号 (Signals) —— 本次新增重点
│       ├── 信号基础
│       │   ├── ● 发送方：内核/其他进程 → 经内核转发 → 目标进程
│       │   └── ● 处理方式：`默认处理` 或 `自定义信号处理函数`
│       ├── ★ 未决信号 & 阻塞信号
│       │   ├── ● `未决信号：已发送但进程尚未接收处理的信号`
│       │   ├── ● `同类型信号排队阻塞`：同类新信号排队等待，不覆盖
│       │   └── ● `信号屏蔽`：可手动设置阻塞，取消阻塞后排队信号才处理
│       ├── ★ `进程两位向量（必考）`
│       │   ├── ● `未决向量 (Pending Vector)`: 位=1 → 对应信号处于未决状态
│       │   └── ● `阻塞向量 (Blocked Vector)`: 位=1 → 对应信号被阻塞
│       └── ★ `信号处理上下文切换（核心难点）`
│           ├── ● 普通跳转：只保存栈/参数/缓冲区，不保存信号屏蔽状态
│           └── ● `信号专用跳转接口`：必须用于信号处理程序中；
│               会同步保存当前信号屏蔽向量，恢复时还原屏蔽状态
│
├── 第九章：虚拟内存
│   ├── 地址概念
│   │   ├── 虚拟地址 (VA)、物理地址 (PA)
│   │   ├── ★ 页表 (Page Table)：VPN → PPN 映射
│   │   └── ● `页表作用`：将虚拟页号映射到物理页号
│   │
│   ├── 地址翻译
│   │   ├── ★ TLB（快表）：加速虚→实地址变换
│   │   ├── ◆ `虚地址结构`：VPN（高位）+ VPO（低位，=PPO）
│   │   ├── ◆ `TLB结构`：TLBT（Tag）+ TLBI（组索引）
│   │   └── ◆ 多级页表
│   │
│   └── ★ `内存分配`（分值占比高）
│       ├── ◆ 堆内存动态分配
│       │   └── `malloc()` / `free()`
│       ├── ★ 内存分配结构体 → 需熟记
│       ├── ◆ `空闲块合并规则`
│       │   └── ● 分配/释放时检查前后相邻块是否空闲 → 满足条件则合并
│       └── ◆ `空闲块检索方式`：首次适配 / 下次适配 / 最佳适配
│
└── 附：Cache 与性能
    ├── Cache 结构参数
    │   ├── `S`（组数）, `E`（每组行数）, `B`（块大小）
    │   └── ◆ `组相联/直接映射/全相联`
    ├── ◆ Cache 命中 / 不命中率计算
    │   ├── 冷不命中 (Cold Miss)
    │   ├── 冲突不命中 (Conflict Miss)
    │   └── 容量不命中 (Capacity Miss)
    └── ◆ 地址划分：CO（块内偏移）+ CI（组索引）+ CT（标记）
```
